# China Fund Research Skill

一个用于研究中国市场基金的 Codex Skill，覆盖公募基金、ETF、LOF、QDII、债券基金、指数基金和主动权益基金。

## 能做什么

- 核对基金代码、类型、策略与业绩基准
- 分析收益质量、波动、回撤和持仓集中度
- 检查基金经理、费用、申赎规则与流动性
- 比较多只基金并生成买入前检查清单
- 明确数据日期、缺失信息与主要风险

## 基于什么进行分析和判断

本 Skill 不预测基金下一交易日的涨跌，而是基于以下信息做研究辅助和风险判断：

1. **基金身份与策略**
   - 基金代码、完整名称、份额类别和基金类型
   - 跟踪指数、业绩比较基准或主动投资策略
   - 主要资产类别、行业和风格暴露

2. **收益与风险特征**
   - 不同时间区间的收益表现
   - 最大回撤、波动率、回撤修复时间和稳定性
   - 与合适基准及同类基金的差异
   - 历史表现仅作为风险特征参考，不作为未来收益保证

3. **持仓与集中度**
   - 前十大持仓、行业集中度和风格偏向
   - 多只基金之间的指数、行业和底层资产重复程度
   - 债券基金的久期、信用风险、杠杆和可转债暴露
   - QDII 的海外市场、汇率、额度和节假日错位风险

4. **管理人与产品运作**
   - 基金经理任职时间、风格稳定性和同类产品表现
   - 基金公司、基金规模、规模变化和运作稳定性
   - 指数基金的跟踪误差，主动基金的持仓与策略一致性

5. **费用与交易摩擦**
   - 管理费、托管费、销售服务费、申购费和赎回费
   - 持有期、限购、暂停申购、到账时间和巨额赎回风险
   - ETF/LOF 的成交量、买卖价差以及溢价或折价

6. **用户自身约束**
   - 投资目标、资金用途和预计持有期限
   - 可接受回撤、仓位上限和现金储备要求
   - 当前持仓、成本、份额和已有行业暴露
   - 用户约束优先于单纯按历史收益进行排序

7. **数据质量与日期**
   - 优先参考基金公司、交易所、基金公告和监管披露
   - 净值、持仓、规模、费率和基金经理数据必须区分日期
   - 盘中估值不等于最终净值；数据缺失或冲突时会明确标注

判断结果通常回答的是：基金是否与目标匹配、组合是否过度集中、交易成本是否合理、当前数据是否足以操作，以及哪些条件满足后才值得进一步考虑。

## 用户需要提供什么数据

### 最低必要信息

分析单只基金时，至少提供：

- 基金代码，最好同时提供完整名称和 A/C 等份额类别
- 想解决的问题，例如买入前检查、持仓诊断、基金对比或风险复盘
- 预计持有期限的大致范围
- 可接受的风险或最大回撤范围

如果只提供基金代码，Skill 仍可进行产品层面的通用研究，但无法准确判断它是否适合用户的实际组合。

### 持仓诊断建议提供

- 每只基金的代码和确认份额，或当前持仓金额
- 持仓成本、持有收益和买入日期/持有天数
- 适用的赎回费阶梯
- 组合总金额、单只基金或组合仓位上限
- 最低现金储备
- 是否有定投、在途申购、待确认赎回或其他未完成交易

### 个性化风险检查建议提供

- 资金用途以及可能使用资金的时间
- 最大可接受回撤，并说明针对单只基金还是整个组合
- 是否允许行业主题基金、海外基金、债券信用风险等特定暴露
- 是否有不能卖出的仓位、税费、平台限制或流动性要求

### 数据提供方式

可以直接提供文字、表格、基金平台截图或脱敏后的持仓文件。不要上传账户密码、身份证号、银行卡号、验证码、完整交易账号等敏感信息。

### 信息不足时如何处理

- 缺少实时数据：标注数据日期，不把旧数据描述为最新数据
- 缺少持仓信息：只做基金产品研究，不推断用户真实仓位
- 缺少费用或持有期：卖出判断以暂缓或小额检查为主
- 缺少风险目标：给出多个情景，不替用户选择
- 数据互相冲突：说明冲突，并优先采用官方或用户确认的数据

## 不做什么

- 不替用户下单
- 不承诺收益或预测短期涨跌
- 不使用“稳赚”“必买”等确定性表达
- 不把历史业绩视为未来收益保证
